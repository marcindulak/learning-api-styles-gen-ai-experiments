APIClient is a strict superset of Client (FR-007 still passes) and clears `_dont_enforce_csrf_checks=False` so DRF's `SessionAuthentication.enforce_csrf` does not reject session-authenticated POSTs from tests. Falls back to `Client` if DRF is not installed, preserving the plain-`behave` path from ENTRY 002.
- City-limit (`SUPPORTED_CITY_LIMIT = 5`) check lives in `CityViewSet.create` rather than the serializer, so the 400 response carries `detail` (a single message) instead of a per-field error dict. The Gherkin asserts `body.detail` containing "city limit reached"; raising in the serializer's `validate` would have produced `body.non_field_errors` instead.
